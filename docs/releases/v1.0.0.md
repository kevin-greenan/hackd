# hackd v1.0.0

The first stable internal release of hackd is ready. This release packages the security training platform runtime for single-host internal Docker Compose deployments.

## Highlights

- Local email/password authentication with server-side learner/admin authorization.
- Learner dashboard, module pages, Markdown lessons, assignments, challenge progress, and completion tracking.
- Admin workflows for users, groups, modules, challenges, assignments, runtime instances, reports, audit logs, attachments, branding, and themes.
- Static flag, short-answer, multiple-choice, file attachment, and simple Dockerized web challenge support.
- Versioned YAML/JSON content schema with import dry-run validation and examples.
- CSV reporting export contracts for completions and attempts.
- CSRF protection for state-changing routes, secure password hashing, signed HTTP-only sessions, and disabled-user login blocking.
- Docker runtime runner with image allowlisting, unprivileged containers, dropped capabilities, no-new-privileges, read-only root filesystems, and CPU, memory, and PID limits.
- Docker Compose deployment docs, readiness and health checks, structured logs, backup/restore scripts, release checklist, UI smoke tests, and container/dependency scanning commands.

## Scope

`v1.0.0` is an internal, single-host Docker Compose release. It is not a public SaaS platform, multi-tenant LMS, or production-grade hostile-workload cyber range.

Local auth and in-memory login/submission throttling are accepted for this v1 scope. OIDC/SAML/MFA/SCIM, distributed rate limiting, password reset, scheduled reports, file-based answer submissions, dynamic flags, Git-backed content repositories, and stronger runtime sandboxing remain post-v1 items.
