# Content Schema V1

hackd content bundles use YAML or JSON with `version: 1`.

Schema V1 is the stable import format for local modules and challenges. The importer validates the full bundle before writing and then upserts modules, challenges, and module-to-challenge links by slug.

## Bundle

```yaml
version: 1
modules: []
challenges: []
```

| Field | Required | Notes |
| --- | --- | --- |
| `version` | yes | Must be `1`. Other versions are rejected. |
| `modules` | yes | At least one module. |
| `challenges` | no | Challenge definitions linked by module challenge entries. Defaults to an empty list. |

## Module

| Field | Required | Notes |
| --- | --- | --- |
| `title` | yes | 2-160 characters. |
| `slug` | yes | Lowercase kebab-case, 2-64 characters. |
| `summary` | yes | 10-500 characters. |
| `bodyMarkdown` | conditional | Inline Markdown body. Use exactly one of `bodyMarkdown` or `bodyFile`. |
| `bodyFile` | conditional | Relative Markdown path inside the content directory. Absolute paths and parent escapes are rejected. |
| `difficulty` | yes | 2-60 characters. |
| `estimatedMinutes` | no | Integer 0 or greater, or `null`. |
| `status` | no | `DRAFT`, `PUBLISHED`, or `ARCHIVED`. Defaults to `DRAFT`. |
| `tags` | no | Up to 20 non-empty tags, each up to 40 characters. |
| `challenges` | no | Module challenge links. Defaults to an empty list. |

## Module Challenge Link

| Field | Required | Notes |
| --- | --- | --- |
| `slug` | yes | Must reference a challenge defined in the same bundle. |
| `sortOrder` | no | Integer 0-10000. Defaults to `0`. |
| `required` | no | Boolean. Defaults to `true`. |

## Challenge

| Field | Required | Notes |
| --- | --- | --- |
| `title` | yes | 2-160 characters. |
| `slug` | yes | Lowercase kebab-case, 2-64 characters. |
| `description` | yes | 5-1000 characters. |
| `type` | yes | `STATIC_FLAG`, `MULTIPLE_CHOICE`, `SHORT_ANSWER`, `FILE_BASED`, or `DOCKER_WEB`. |
| `difficulty` | yes | 2-60 characters. |
| `points` | no | Integer 0-10000. Defaults to `0`. |
| `status` | no | `DRAFT`, `PUBLISHED`, or `ARCHIVED`. Defaults to `DRAFT`. |
| `tags` | no | Up to 20 tags. |
| `validationConfig` | no | Type-specific JSON object or `null`. |
| `runtimeConfig` | no | Runtime-specific JSON object or `null`. |

## Supported Validation Configs

### Static Flag

```yaml
type: STATIC_FLAG
validationConfig:
  type: static_flag
  flag: flag{example}
```

### Short Answer

```yaml
type: SHORT_ANSWER
validationConfig:
  type: short_answer
  acceptedAnswers:
    - prepared statement
    - parameterized query
  caseSensitive: false
```

### Multiple Choice

```yaml
type: MULTIPLE_CHOICE
validationConfig:
  type: multiple_choice
  allowMultiple: true
  options:
    - id: request-id
      label: Request ID from the user
    - id: server-key
      label: Server-side secret key
  correctOptionIds:
    - request-id
```

### File-Based Challenge

`FILE_BASED` challenges use downloadable attachments managed in the admin UI. Schema V1 can define the challenge metadata and answer validation, but it does not import attachment files yet.

```yaml
type: FILE_BASED
validationConfig:
  type: static_flag
  flag: flag{review-complete}
```

### Dockerized Web Challenge

```yaml
type: DOCKER_WEB
validationConfig:
  type: static_flag
  flag: flag{docker-web}
runtimeConfig:
  type: docker_web
  image: nginx:alpine
  containerPort: 80
  memoryMb: 128
  cpuCount: 0.25
  ttlMinutes: 30
  env: {}
```

The image must match `RUNTIME_ALLOWED_IMAGES` at launch time.

## Import Behavior

- Modules are upserted by `slug`.
- Challenges are upserted by `slug`.
- Module challenge links are upserted by module/challenge pair.
- Omitted modules, challenges, and links are not deleted.
- Existing local edits to omitted content are preserved.
- Imported body Markdown replaces the module body for matching slugs.
- Imported validation/runtime config replaces the matching challenge config for matching slugs.

## Validation Failures

The importer fails before writing when:

- `version` is not `1`.
- A slug is malformed.
- Module slugs are duplicated.
- Challenge slugs are duplicated.
- A module links to a challenge not defined in the bundle.
- A module links to the same challenge more than once.
- A module defines both `bodyMarkdown` and `bodyFile`, or neither.
- `bodyFile` is absolute or escapes the content directory.
- A field violates bounds for text length, points, tags, status, or type.
